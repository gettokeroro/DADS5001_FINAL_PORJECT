ประเด็นแบบ A — SVG Chanseyแบบ B — Emoji + Gradient
หน้าตาวาดทั้งตัว ตา 2 ตา หมวก ปก fluffy — ดูเหมือน chansey ที่ user อยาก refหน้า fallback เป็น emoji แต่ละ OS แสดงต่างกัน (iOS vs Windows)
CodeSVG ยาว ~50 บรรทัด/ตัว → 100 บรรทัดรวมCSS + emoji สั้น ~20 บรรทัด
ปรับแก้ภายหลังแก้ SVG ต้องเข้าใจ vector pathเปลี่ยน emoji ได้ทันที
ทำงานใน Streamlitมีอยู่แล้วใน styling.py แต่ user รู้สึกหน้าตาดูแปลกง่าย เพราะแค่ inline HTML
เหมาะกับ ref Chanseyใกล้เคียงที่สุด — เพราะ chansey คือ vector ตัวกลม ตาโต ใส่หมวกไกลกว่าเพราะ emoji ไม่ chansey-fluffy
คำแนะนำของผม: ใช้ แบบ A (SVG Chansey-style) เพราะใกล้กับ ref Chansey ของพี่มากที่สุด และโค้ดมีอยู่ใน utils/styling.py อยู่แล้ว — แค่ปรับ render_nurse_mascot/render_doctor_mascot ให้แสดงในกล่อง pink/blue ที่มีชื่อ + คำพูดข้างใต้ (เหมือนใน mockup v4)

ถ้าใน Streamlit หน้าตาดูแปลก อาจเป็นเพราะ:
  • SVG ปัจจุบันถูกใส่เข้าไปแบบ horizontal layout (mascot อยู่ซ้าย speech อยู่ขวา) ทำให้ดูไม่ค่อยเด่น
  • ขนาด SVG เล็กไป (88px) — mockup v4 ใช้ ~140px แล้วใส่ใน card pink มี border
